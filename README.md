# Slack移行チェックツール 操作ガイド

このツールは、Slack旧ワークスペース（移行元）と新ワークスペース（移行先）のメンバーを照合し、
移行漏れやメールアドレスルール違反をスプレッドシートで確認するための Google Apps Script です。

## このツールでできること

- 旧WSユーザーごとに、新WSへの移行有無を判定
- 移行済みユーザーのメールアドレスがルールに合っているかを判定
- 旧WSと照合できない新WSユーザーを抽出
- 判定から除外したいユーザー（既存メンバーなど）を除外リストで管理

除外リストに入っているメールアドレスは、旧WS・新WSの両方の判定から除外されます。

## 1. 事前に準備するもの

### 1-1. スプレッドシート

以下3つのシートを用意します。

- `除外リスト`（A列: 氏名 / B列: メアド）
- `名前修正ルール`（A列: 旧WS氏名 / B列: 新WS氏名 / C列: 説明）
- `移行確認結果`（実行結果の出力先。毎回上書き）

`除外リスト` の1行目はヘッダーにしてください。

- A1: 氏名
- B1: メアド

`名前修正ルール` は実行時に自動作成されます（最初は空で問題ありません）。

### 1-2. Slackトークン

旧WS用と新WS用のトークンを用意します。

- `SLACK_TOKEN_OLD`
- `SLACK_TOKEN_NEW`

必要スコープ:

- `users:read`
- `users:read.email`

## 2. 初回セットアップ手順（メニュー位置つき）

### 手順1: Apps Scriptエディタを開く

1. 対象スプレッドシートを開く
2. 上部メニュー `拡張機能 > Apps Script` をクリック

## 3. 実行手順（定常運用）

### 手順1: 本チェックを実行

1. スプレッドシート上部メニュー `Slack移行チェック` をクリック
2. `移行チェックを実行` をクリック

### 補足: 除外リストの更新

既存の新WS参加者を除外リストに追加したい場合は、Apps Scriptエディタのコンソールから以下を実行してください。

```javascript
exportCurrentNewUsersToExclusionList();
```

初回のみ認可ダイアログが出るため、画面案内に従って許可してください。
3. 完了後、スプレッドシートの `移行確認結果` を開く

## 4. 結果の見方

`移行確認結果` の `判定ステータス` は次の4種類です。

- `移行完了（正常）`
  - 旧WSと新WSで照合でき、かつ新WSメールがルールに合致
- `メアド不正`
  - 旧WSと新WSで照合できたが、新WSメールがルール外
- `未移行`
  - 旧WSユーザーに対応する新WSユーザーが見つからない
- `照合不可（新WSのみ存在）`
  - 新WSユーザーだが、旧WS側に照合先がない

出力結果は、次の順で自動的に並び替えられます。

1. `未移行`
2. `メアド不正`
3. `照合不可（新WSのみ存在）`
4. `移行完了（正常）`

また、行の背景色でも状態を見分けやすくしています。

- `未移行` と `メアド不正`: 赤系
- `照合不可（新WSのみ存在）`: 黄系
- `移行完了（正常）`: 薄緑系

## 5. 判定ルールの補足

- 氏名は正規化して照合します（空白除去など）
- 同姓同名がいる場合は1名ずつ割り当てて判定します
- メールアドレスの判定は `MAIL_RULE_SUFFIX` で行います
  - 現在値: `17th.fw@gmail.com`

## 6. 名前修正ルール（氏名ゆれの対応）

旧WSと新WSで氏名の表記が異なる場合（「山田太郎」vs「山田太朗」など）、
`名前修正ルール` シートで修正ルールを登録することで、自動的に正しく照合できるようになります。

### 使い方

1. `移行確認結果` で「未移行」または「照合不可」と表示されたユーザーを確認
2. 同じ人物だと判定した場合、`名前修正ルール` シートに以下のように入力
   - A列（旧WS氏名）：`山田太郎`
   - B列（新WS氏名）：`山田太朗`
   - C列（説明）：修正理由など（任意）
3. 次回の実行時から、修正ルールが自動的に適用され、正しく照合されます

### 注意点

- 修正ルールシートに入力した内容は、スクリプト再実行後も保持されます
- 修正ルールが不要になったら、該当行を削除してください
- 修正ルールなしで運用することも可能です（空のままでも問題ありません）

## 7. トラブル時のチェックリスト

- `Script property is missing: ...` が出る
  - Script Properties のキー名に誤字がないか確認
- Slack APIエラーが出る
  - トークン権限に `users:read` / `users:read.email` があるか確認
- `Sheet not found: ...` が出る
  - シート名が `除外リスト` / `移行確認結果` と完全一致か確認

## 8. 運用のコツ

- 最初に `Slack移行チェック > 除外リストを新WSユーザーで更新` を実行して除外対象を整理しておくと、ノイズが減って確認しやすくなります
- 氏名ゆれが多い場合は、`名前修正ルール` シートで修正ルールを登録して対応します
- `移行確認結果` は判定ステータス順に自動整列されるため、上から順に未対応を追うと確認しやすくなります
