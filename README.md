# Slack移行チェックツール 操作ガイド（スクリーンショット対応版）

このツールは、Slack旧ワークスペース（移行元）と新ワークスペース（移行先）のメンバーを照合し、
移行漏れやメールアドレスルール違反をスプレッドシートで確認するための Google Apps Script です。

## このツールでできること

- 旧WSユーザーごとに、新WSへの移行有無を判定
- 移行済みユーザーのメールアドレスがルールに合っているかを判定
- 旧WSと照合できない新WSユーザーを抽出
- 判定から除外したいユーザー（既存メンバーなど）を除外リストで管理

## 0. このガイドの使い方

- このREADMEは、画面キャプチャを貼りながら社内共有できる構成です
- `【スクショX】` の箇所に画像を貼ってください
- クリック位置が分かるよう、メニュー階層を明記しています

## 1. 事前に準備するもの

### 1-1. スプレッドシート

以下2つのシートを用意します。

- `除外リスト`（A列: 氏名 / B列: メアド）
- `移行確認結果`（実行結果の出力先。毎回上書き）

`除外リスト` の1行目はヘッダーにしてください。

- A1: 氏名
- B1: メアド

### 1-2. Slackトークン

旧WS用と新WS用のトークンを用意します。

- `SLACK_TOKEN_OLD`
- `SLACK_TOKEN_NEW`

必要スコープ:

- `users:read`
- `users:read.email`

## 2. 初回セットアップ手順（メニュー位置つき）

### 手順1: Apps Scriptエディタを開く

1. 対象スプレッドシートを開く
2. 上部メニュー `拡張機能 > Apps Script` をクリック

【スクショ1】スプレッドシートの `拡張機能 > Apps Script` を開く画面

### 手順2: Script Properties にトークンを登録

1. Apps Scriptエディタ左側の `プロジェクトの設定`（歯車アイコン）をクリック
2. `スクリプト プロパティ` セクションで `スクリプト プロパティを追加`
3. 以下2件を登録して保存

- キー: `SLACK_TOKEN_OLD` / 値: 旧WSトークン
- キー: `SLACK_TOKEN_NEW` / 値: 新WSトークン

【スクショ2】`プロジェクトの設定` 画面で Script Properties を追加する画面

### 手順3: シート名を確認

スプレッドシート下部のタブ名が以下と一致していることを確認します。

- `除外リスト`
- `移行確認結果`

【スクショ3】シートタブ名の確認画面

### 手順4: スプレッドシートを再読み込みしてメニューを表示

1. スプレッドシート画面に戻る
2. ブラウザ再読み込み（またはシート再オープン）
3. 上部メニューバーに `Slack移行チェック` が表示されることを確認

【スクショ4】メニューバーに `Slack移行チェック` が表示された画面

## 3. 実行手順（定常運用）

### 手順1: 必要なら除外リストを自動作成・更新

既存の新WS参加者を除外したい場合は、先に次のメニューを実行します。

1. スプレッドシート上部メニュー `Slack移行チェック` をクリック
2. `除外リストを新WSユーザーで更新` をクリック

初回のみ認可ダイアログが出るため、画面案内に従って許可してください。

【スクショ5】`Slack移行チェック > 除外リストを新WSユーザーで更新` の実行画面

### 手順2: 本チェックを実行

1. スプレッドシート上部メニュー `Slack移行チェック` をクリック
2. `移行チェックを実行` をクリック
3. 完了後、スプレッドシートの `移行確認結果` を開く

【スクショ6】`Slack移行チェック > 移行チェックを実行` の実行画面
【スクショ7】`移行確認結果` シートの出力例

## 4. 結果の見方

`移行確認結果` の `判定ステータス` は次の4種類です。

- `移行完了（正常）`
  - 旧WSと新WSで照合でき、かつ新WSメールがルールに合致
- `メアド不正`
  - 旧WSと新WSで照合できたが、新WSメールがルール外
- `未移行`
  - 旧WSユーザーに対応する新WSユーザーが見つからない
- `照合不可（新WSのみ存在）`
  - 新WSユーザーだが、旧WS側に照合先がない

## 5. 判定ルールの補足

- 氏名は正規化して照合します（空白除去など）
- 同姓同名がいる場合は1名ずつ割り当てて判定します
- メールアドレスの判定は `MAIL_RULE_SUFFIX` で行います
  - 現在値: `17th.fw@gmail.com`

## 6. トラブル時のチェックリスト

- `Script property is missing: ...` が出る
  - Script Properties のキー名に誤字がないか確認
- Slack APIエラーが出る
  - トークン権限に `users:read` / `users:read.email` があるか確認
- `Sheet not found: ...` が出る
  - シート名が `除外リスト` / `移行確認結果` と完全一致か確認

## 7. 運用のコツ

- 最初に `Slack移行チェック > 除外リストを新WSユーザーで更新` を実行して除外対象を整理しておくと、ノイズが減って確認しやすくなります
- 氏名ゆれが多い場合は、結果を見ながら `除外リスト` を調整して再実行すると精度が上がります
